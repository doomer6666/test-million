# Список 1,000,000 элементов

Fullstack приложение для работы со списком из миллиона элементов: выбор, фильтрация, сортировка drag&drop и инфинити-скролл.

## Стек

- **Backend**: Node.js + Express
- **Frontend**: React 18 + Vite + @dnd-kit

## Архитектурные решения

### Хранилище (без БД)

- В памяти сервера хранится `Set` выбранных ID и массив `selectedOrder` для сохранения порядка.
- Начальные элементы 1..1_000_000 не хранятся как объекты — ID генерируются на лету, что экономит память (~50MB вместо ~200MB).
- Добавленные пользователем элементы с новыми ID хранятся в `Map`.

### Очередь запросов с дедупликацией

Все мутирующие запросы проходят через `Batcher`:

- Дедупликация по ключу операции: если в буфере уже есть запрос с тем же ключом — он заменяется (последний выигрывает).
- Это значит, например, что пять кликов «добавить ID=42» за секунду превратятся в одну операцию.
- Батчинг по типу операции:
  - добавление элементов — буфер 10 сек;
  - получение/изменение — буфер 1 сек.

### Drag&Drop с фильтрацией

Для корректной работы drag&drop в отфильтрованном списке клиент при инициализации загружает **полный порядок** ID (`GET /api/selected/all`) и далее фильтрует/пагинирует локально. При drop обновляется весь массив и отправляется на сервер.

## Запуск

### Backend

```bash
cd backend
npm install
npm run dev
# Сервер на http://localhost:3001
```

### Frontend

```bash
cd frontend
npm install
npm run dev
# Клиент на http://localhost:3000
```

Откройте http://localhost:3000 — Vite проксирует `/api/*` на бэкенд.

## API

| Метод  | Путь                                   | Описание                                    |
| ------ | -------------------------------------- | ------------------------------------------- |
| GET    | `/api/items?search=&offset=&limit=`    | Невыбранные элементы (пагинация, фильтр)    |
| GET    | `/api/items/count?search=`             | Кол-во невыбранных по фильтру (для UI)      |
| POST   | `/api/items`                           | Добавить новый элемент `{id}` (батч 10с)    |
| GET    | `/api/selected?search=&offset=&limit=` | Выбранные элементы (пагинация, фильтр)      |
| GET    | `/api/selected/all`                    | Полный порядок ID выбранных (для drag&drop) |
| POST   | `/api/selected`                        | Выбрать элемент `{id}` (батч 1с)            |
| DELETE | `/api/selected/:id`                    | Убрать из выбора (батч 1с)                  |
| PUT    | `/api/selected/order`                  | Новый порядок `{order: []}` (батч 1с)       |
| GET    | `/api/stats`                           | Сводка: всего/выбрано/добавлено             |

## Особенности реализации

- **Поиск не сохраняется** между перезагрузками страницы — он локальный.
- **Состояние выбора и порядка** хранится на сервере, общий для всех посетителей.
- При перезагрузке страницы порядок восстанавливается автоматически.
- Debounce поиска на клиенте (300мс) + серверный батчер не дублируют запросы.
- Drag&drop работает в пределах загруженных элементов. Если вы хотите переставить что-то в «далёкую» позицию — просто проскролльте, чтобы подгрузились соседние элементы.
